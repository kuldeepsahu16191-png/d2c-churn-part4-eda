# Model Monitoring & Responsible-Use Plan

This document outlines the operational monitoring framework and ethical guardrails for deploying the D2C Customer Churn Prediction API service.

---

## 📈 1. Operational Monitoring Plan

To ensure long-term model reliability and capture performance degradation over time, the API service tracks telemetry across four key areas:

### Data and Concept Drift
* **Feature Drift:** We monitor production input feature distributions (such as `recency`, `monetary_value`, and `support_complaints`) against the training baseline cohort using a rolling two-sample Kolmogorov-Smirnov (KS) test.
* **Prediction Distribution Drift:** The daily ratio of predicted churn tags (Class 1 vs Class 0) is logged. A sudden spike or dip in predicted churn signifies potential shifts in macroeconomic behavior or data collection pipelines.

### System Metrics
* **API Performance:** Track API endpoint response latency (p95 and p99 metrics) to ensure internal CRM tools do not face integration delays.
* **Error Logging:** Monitor the frequency of HTTP 422 (Pydantic validation failures) and HTTP 500 (internal server/inference errors) codes.

### Retraining Triggers
The model will be scheduled for a retraining pipeline if any of the following parameters are met:
1.  **Performance Drop:** True validation precision or recall slips by more than 10% when evaluated against actual 60-day downstream customer retention outcomes.
2.  **Statistical Feature Drift:** Any core behavioral feature displays significant data drift with a calculated p-value dropping below $0.05$.
3.  **Time Elapsed:** A hard scheduling threshold requires a complete model refresh every 90 days to natively capture seasonal updates in personal care purchasing patterns.

## 🛑 2. Responsible AI Usage Policy

The outputs generated by this API must be used ethically to enhance customer relationships, rather than taking punitive actions.

### Approved Applications (Do's)
* **Proactive Support Escalation:** Prioritizing customer support tickets and live chats for accounts flagged with a high churn probability and active complaints.
* **Value-Driven Outreach:** Offering tailored product recommendations, early access to new personal care product launches, or curated loyalty rewards to rebuild brand sentiment.
* **Experience Personalization:** Dynamically adjusting onboarding paths or app/web experiences for at-risk users to increase engagement.

### Explicit Restrictions (Don'ts)
* **Universal Blind Discounting:** Do not use prediction flags to push generic discount codes to all at-risk users. This damages margins and trains healthy customers to change habits just to trigger markdowns.
* **Account Restrictions:** High churn risk scores must never be used to freeze accounts, delay delivery priority, or reduce user access tiers.
* **Predatory Pricing Schemes:** The API outputs must never feed dynamic optimization frameworks that raise pricing variables or delivery fees on groups flagged as vulnerable or highly likely to churn.
